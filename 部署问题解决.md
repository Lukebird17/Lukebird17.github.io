# GitHub Pages 部署问题解决方案

## ❌ 错误信息
```
Branch "main" is not allowed to deploy to github-pages due to environment protection rules.
The deployment was rejected or didn't satisfy other protection rules.
```

## ✅ 解决方法

### 方法一：调整环境保护规则（推荐）

#### 第 1 步：进入仓库设置
1. 打开你的 GitHub 仓库：`https://github.com/Lukebird17/Lukebird17.github.io`
2. 点击 **Settings**（设置）

#### 第 2 步：配置 Pages 源
1. 在左侧菜单找到 **Pages**
2. 在 **Build and deployment** 部分
3. **Source** 下拉菜单选择：**GitHub Actions**
   - ⚠️ 不要选择 "Deploy from a branch"
4. 保存后会自动应用

#### 第 3 步：配置环境保护规则
1. 在左侧菜单找到 **Environments**
2. 点击 **github-pages** 环境
3. 找到 **Deployment branches and tags** 部分
4. 两个选择：
   - **选项 A（推荐）**: 选择 **All branches**
   - **选项 B**: 选择 **Selected branches and tags** → 点击 **Add deployment branch or tag rule** → 输入 `main`
5. 如果有 **Required reviewers**（需要审查者），暂时取消勾选
6. 点击 **Save protection rules**

#### 第 4 步：配置工作流权限
1. **Settings** → **Actions** → **General**
2. 滚动到底部，找到 **Workflow permissions**
3. 选择 **Read and write permissions**
4. 勾选 **Allow GitHub Actions to create and approve pull requests**
5. 点击 **Save**

#### 第 5 步：重新触发部署

**方法 A：通过 Push 触发**
```bash
cd /Users/leon/Project/personal-homepage

# 做一个小改动
git commit --allow-empty -m "Trigger deployment"
git push
```

**方法 B：手动触发**
1. 进入仓库的 **Actions** 标签
2. 点击左侧的 **Deploy Hugo site to GitHub Pages**
3. 点击右侧的 **Run workflow** 按钮
4. 选择 `main` 分支
5. 点击绿色的 **Run workflow** 按钮

### 方法二：简化工作流配置

如果方法一不行，可以尝试简化工作流：

#### 修改 `.github/workflows/hugo.yml`

将 `deploy` job 中的 `environment` 部分改为：

```yaml
deploy:
  environment:
    name: github-pages
  runs-on: ubuntu-latest
  needs: build
  permissions:
    pages: write
    id-token: write
  steps:
    - name: Deploy to GitHub Pages
      id: deployment
      uses: actions/deploy-pages@v4
```

### 方法三：检查仓库可见性

确保仓库是 **Public**（公开）：
1. **Settings** → **General**
2. 滚动到底部 **Danger Zone**
3. 确认仓库是 **Public**
4. 如果是 Private，需要 GitHub Pro 才能使用 GitHub Pages

## 🔍 常见问题排查

### 问题 1：Actions 没有运行
- 检查 **Actions** 标签是否启用
- **Settings** → **Actions** → **General** → **Actions permissions**
- 选择 **Allow all actions and reusable workflows**

### 问题 2：权限不足
- 确保工作流有 `pages: write` 和 `id-token: write` 权限
- 确保 Settings → Actions → Workflow permissions 设置正确

### 问题 3：环境不存在
- GitHub 会自动创建 `github-pages` 环境
- 如果不存在，运行一次工作流会自动创建

### 问题 4：分支保护规则冲突
- **Settings** → **Branches**
- 检查 `main` 分支是否有限制性的保护规则
- 可以暂时禁用来测试

## ✅ 验证部署成功

1. 进入 **Actions** 标签
2. 查看最新的工作流运行
3. 应该看到绿色勾号 ✓
4. 访问 `https://Lukebird17.github.io` 查看网站

## 📞 如果还是不行

### 最后的备用方案：删除并重新创建环境

1. **Settings** → **Environments**
2. 点击 **github-pages** 旁边的 **...** 
3. 选择 **Delete environment**
4. 确认删除
5. 重新 push 代码，GitHub 会自动创建新环境

## 🎯 完整配置检查清单

- [ ] Settings → Pages → Source = **GitHub Actions**
- [ ] Settings → Environments → github-pages → Deployment branches = **All branches** 或包含 `main`
- [ ] Settings → Actions → Workflow permissions = **Read and write permissions**
- [ ] 仓库必须是 **Public**
- [ ] Actions 已启用
- [ ] 工作流文件在 `.github/workflows/` 目录下
- [ ] `main` 分支存在并已推送

完成所有设置后，重新 push 或手动触发工作流即可！
